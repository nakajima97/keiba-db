# 要件定義書：購入馬券登録画面

## インテント分析

| 項目 | 内容 |
|---|---|
| ユーザー要求 | 購入馬券を登録する画面を作成して |
| 要求タイプ | 新機能（New Feature） |
| スコープ見積もり | 複数コンポーネント（DB・API・フロントエンド） |
| 複雑度 | Moderate（複数エンティティ、find-or-create ロジック、連続登録UX） |

---

## 機能要件

### FR-01: 購入馬券の登録
ユーザーは以下の情報を入力して購入馬券を登録できる。

| フィールド | 必須 | 説明 |
|---|---|---|
| 開催日 | ○ | 日付ピッカーで入力 |
| レース場 | ○ | 既存レース場から選択 |
| レース番号 | ○ | 整数（1〜12など） |
| 馬券種別 | ○ | 単勝/複勝/馬連/馬単/ワイド/三連複/三連単 |
| 馬番 | ○ | テキスト入力（例: "1"、"1-3"、"1-3-5"） |
| 購入金額 | ○ | 整数（円） |

### FR-02: 連続登録（続けて入力）
- 登録成功後、フォームはリセットされ **レース番号のみ1インクリメント** された状態になる
- 開催日・レース場は直前の値を保持する
- これにより同じ開催の複数レース馬券を素早く連続入力できる
- 例：東京1Rを登録 → 東京2Rの状態でフォームが準備される

### FR-03: 払い戻し金額の登録
- 登録済みの購入馬券に対して払い戻し金額を入力・保存できる
- 払い戻し金額は後から修正できる（当日レース終了後に入力するユースケース）

### FR-04: 登録済み馬券の表示
- フォーム下部に当日セッション中に登録した購入馬券一覧を表示する
- 各馬券に対して払い戻し金額の入力欄を提供する

### FR-05: 関連エンティティの自動作成
- フォーム送信時、バックエンドで以下を自動 find-or-create する:
  - 開催（レース場 + 開催日の組み合わせ）
  - レース情報（開催 + レース番号）
  - 詳細情報（クラス、コース、馬場状態）は省略可能（NULL許容）

---

## 非機能要件

### NFR-01: バリデーション
- フォーム送信前にクライアントサイドバリデーションを行う
- サーバーサイドでも同様のバリデーションを実施（二重バリデーション）
- エラーメッセージはフォームフィールドの下に表示

### NFR-02: パフォーマンス
- フォーム送信からリセット完了まで2秒以内

### NFR-03: セキュリティ（SECURITY拡張 適用）
- SECURITY-05: すべてのAPI入力パラメータにバリデーション
- SECURITY-08: 認証済みユーザーのみアクセス可（auth middleware必須）
- SECURITY-05準拠: LaravelのFormRequestを使用したサーバーサイドバリデーション
- SECURITY-09: エラーレスポンスにスタックトレースを含めない

### NFR-04: プロパティベーステスト（PBT拡張 適用）
- PBT-09: TypeScript用にfast-checkフレームワークを採用
- PBT-01: Functional Designで各コンポーネントのテスト可能プロパティを特定
- PBT-03: 金額・馬番などのバリデーション不変条件をPBTでカバー

## 技術要件

### フロントエンド
- React + TypeScript（Inertia.js）
- FSD（Feature-Sliced Design）アーキテクチャに準拠
- 新規スライス: `features/add-ticket/`, `features/record-payout/`
- 新規エンティティ: `entities/ticket/`, `entities/venue/`, `entities/race/`
- ページコンポーネント: `pages/tickets/create.tsx`

### バックエンド
- Laravel Controller + FormRequest
- Eloquent Model（Venue, Meeting, Race, Ticket, Payout）
- マイグレーション: 5テーブル新規作成
- 認証: auth + verified ミドルウェア適用

### データモデル（新規テーブル）
| テーブル名 | エンティティ | 主な列 |
|---|---|---|
| venues | レース場 | name, short_name, organizer_type |
| meetings | 開催 | venue_id, date |
| races | レース情報 | meeting_id, race_number, class, surface_condition |
| tickets | 購入馬券 | race_id, bet_type, amount, horse_numbers |
| payouts | 払い戻し | ticket_id, payout_amount |
