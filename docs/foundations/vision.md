# Vision Document

## エグゼクティブサマリー

競馬の収支記録・レース情報管理・予想分析を行う個人向け Webアプリケーション。
ユーザー自身が購入した馬券と払い戻し金額を記録し、収支を集計・把握することを主目的とする。
加えて、馬・騎手・コース・レース名ごとにメモを残すことで、次回以降の予想に活用できる。

## 解決する問題

競馬の収支を手元で正確に把握する手段がなく、感覚的な管理になりがちである。
また、馬・騎手・コースに関する個人的な知見がメモアプリや記憶に散在しており、
予想時に参照しにくい状態になっている。

## ユーザーペルソナ

- **唯一のユーザー**: 開発者自身（個人利用）
  - PC・スマートフォン両方から利用する
  - 競馬のレース後に馬券・払い戻しを記録する
  - 予想前にメモを参照する

## 成功基準

- 収支記録機能・収支集計機能がデプロイされ、実際に使えること
- PC・スマートフォンどちらからもストレスなく操作できること

## フル機能ビジョン（将来）

| 領域 | 内容 |
|------|------|
| 収支管理 | 購入馬券・払い戻し記録、期間別・レース場別等での収支集計 |
| レース管理 | 開催・レース情報・レース結果の記録 |
| マスター管理 | レース場・馬・騎手・レース名・コースの管理 |
| メモ機能 | 馬・騎手・コース・レース名・レース場への自由メモ記録 |
| 予想分析 | 記録データを元にした傾向分析（将来検討） |

## MVP 対象機能

| 機能 | 理由 |
|------|------|
| 認証（ログイン・ログアウト） | 個人データ保護のために必須 |
| 収支集計画面 | 最優先の目的 |
| 開催管理（CRUD） | レース情報の親エンティティとして必要 |
| レース情報管理（CRUD） | 購入馬券の紐付け先として必要 |
| 購入馬券管理（CRUD） | 収支記録の中核 |
| 払い戻し管理（CRU、削除なし） | 収支計算に必要、削除は馬券削除に連動 |
| レース場管理（CRUD） | 開催の紐付け先として必要 |
| 馬・騎手・コース・レース名管理（CRUD） | レース結果・メモの紐付け先として必要 |
| 各マスターへのメモ機能 | 予想活用の中核 |
| レース結果管理（CRUD） | 馬・騎手の実績記録 |
| レース公式払戻金管理（CRU） | 参考情報として記録 |

## MVP 対象外機能

| 機能 | 理由 |
|------|------|
| 外部データ連携（JRA-VAN等） | 費用が発生するため。データはコピペで手入力 |
| スクレイピングによるデータ取得 | 対策されているため実施しない |
| 予想AI・機械学習機能 | 将来検討 |
| 複数ユーザー対応 | 個人利用のため不要 |
| データエクスポート機能 | 初期リリースでは不要 |

## リスクと未確定事項

| 項目 | 内容 |
|------|------|
| データ管理 | 競馬データは個人情報ではないが、記録データはリポジトリにコミットしない |
| データ入力コスト | 手入力・コピペが前提のため、入力フォームのUXが重要 |
| デプロイ環境 | レンタルサーバー上での Node.js ビルドは不可のため、GitHub Actions でビルドしてデプロイ |
