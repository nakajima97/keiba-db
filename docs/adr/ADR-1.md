# ADR-1: 払い戻し削除機能を実装しない

## Status
Accepted

## Context
払い戻し金額の削除機能（D）をタスク一覧に含めるかどうかを検討した。
払い戻し金額は購入馬券に紐づく付随情報であり、単独で削除するユースケースがあるか議論になった。

## Decision
払い戻し削除機能は実装しない。理由は以下の2点：

1. 購入馬券を削除した際に払い戻し金額も連動削除される
2. 払い戻し額を0円と記録することで、実質的に削除と同等の状態を表現できる

## Consequences
### メリット
- 機能をシンプルに保てる
- 誤削除のリスクがない

### デメリット
- 払い戻し金額を0円と記録したレコードがデータとして残るため、集計時に除外条件が必要になる場合がある
